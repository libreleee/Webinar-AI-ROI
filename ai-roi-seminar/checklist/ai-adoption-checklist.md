# AI 도입 체크리스트 (AI ROI 관점)

> 목적: “일단 해보자”를 방지하고,  
> PoC → 운영 → 확산까지 ROI가 가능한 구조인지 빠르게 점검한다.

---

## 1. 문제 정의 & KPI/ROI
- [ ] AI로 해결할 비즈니스 문제가 1~3개로 명확히 정의되어 있는가?
- [ ] KPI가 수치로 정의되어 있는가? (예: 응답시간 -30%, 불량률 -20%)
- [ ] ROI 산정 방식(절감/증대/생산성)이 합의되어 있는가?
- [ ] 성공 기준(Go) / 중단 기준(No-Go)이 있는가?

---

## 2. 경영진 스폰서십 & 의사결정
- [ ] C-level 스폰서가 지정되어 있는가?
- [ ] 조직 저항/우선순위 충돌을 조정할 권한이 있는가?
- [ ] 예산(인력/데이터/인프라/운영)이 확보되어 있는가?

---

## 3. 인력(“자격 있는 담당자” 여부)
- [ ] AI/ML 엔지니어(모델/평가/한계 이해)가 참여하는가?
- [ ] DB 엔지니어(모델링/품질/성능)가 참여하는가?
- [ ] 데이터 엔지니어(파이프라인/스트리밍)가 참여하는가?
- [ ] SI 인력(API/연동/아키텍처)이 참여하는가?
- [ ] SM/운영 인력(ITIL/SLA/모니터링)이 참여하는가?

---

## 4. 데이터 준비(학습/검증/테스트)
- [ ] 학습 데이터가 충분한가? (양·다양성·대표성)
- [ ] 데이터 품질 이슈(결측/중복/오류)에 대한 처리 방식이 있는가?
- [ ] 라벨링/정답 데이터(지도학습)의 확보 계획이 있는가?
- [ ] 편향(Bias) 점검 계획이 있는가?

---

## 5. RTE / ETL / 데이터 파이프라인
- [ ] 업무시스템에 부하 없이 데이터 수집 가능한가? (CDC/이벤트 기반)
- [ ] 실시간/준실시간 스트림(Kafka/Pulsar 등)이 필요한가?
- [ ] ETL/ELT가 자동화되어 있는가?
- [ ] Feature Store 또는 피처 재사용 체계가 있는가?

---

## 6. 인프라 & 비용(GPU/클라우드)
- [ ] PoC 비용과 운영 비용을 분리해 산정했는가?
- [ ] GPU 수요(학습/추론)와 병목을 계산해봤는가?
- [ ] 비용 폭증 방지(Quota/Rate limit/캐시) 설계가 있는가?

---

## 7. PoC(8~12주) 설계
- [ ] PoC 기간이 제한되어 있는가?
- [ ] PoC에서 “운영 전환 가능성”까지 검증하는가?
- [ ] 성능(정확도) + 비용 + 사용자 경험(현업 적용성)을 함께 평가하는가?

---

## 8. 운영(MLOps + DevOps + ITSM)
- [ ] 모델/데이터 버전관리 및 배포 프로세스가 있는가?
- [ ] Drift 모니터링과 재학습(자동/반자동) 전략이 있는가?
- [ ] SLA/SLO/장애 대응(Incident/Problem/Change)이 정의되어 있는가?
- [ ] 로그/모니터링/APM/감사 추적이 준비되어 있는가?

---

## 9. 보안·규제·윤리
- [ ] 개인정보/민감정보 처리 기준이 있는가?
- [ ] 접근통제/권한관리/키관리 체계가 있는가?
- [ ] 모델 편향/환각/오작동 대응 프로세스가 있는가?

---

## 최종 판단
- [ ] 지금 상태는 “실험”이 아니라 “운영 가능한 AI 정보시스템”으로 가는 구조인가?
- [ ] “준비된 실행”인가, “일단 해보자”인가?
